## Directory Purposes

**`content/`:**
- Purpose: Source-of-truth Markdown for everything human-readable on the site and in the CV.
- Contains: Pages, posts (as leaf bundles), TOML frontmatter.
- Key files: `content/_index.md` (homepage), `content/about/_index.md`, `content/previous-work.md`, `content/posts/<slug>/index.md`.

**`themes/terminal/`:**
- Purpose: Vendored Hugo theme (panr/hugo-theme-terminal) consumed as a git submodule.
- Contains: Default templates, partials, shortcodes, base CSS/SCSS.
- Key files: Do not edit in place — override via project `layouts/` / `assets/`.

**`layouts/`:**
- Purpose: Project-level template overrides and additions; takes precedence over the theme.
- Contains: Templates (homepage `index.html`, `404.html`), partials, render hooks, shortcodes.
- Key files: `layouts/index.html`, `layouts/404.html`, `layouts/partials/extended_head.html`, `layouts/partials/structured_data.html`, `layouts/_default/_markup/render-link.html`, `layouts/shortcodes/x.html`.

**`assets/`:**
- Purpose: Inputs for Hugo's asset pipeline (Sass, asset bundling).
- Contains: CSS extensions auto-included by the terminal theme (`resources.Match "css/*.css"`).
- Key files: `assets/css/z-base.css` (shared tokens/animations), `assets/css/z-home.css`, `assets/css/z-404.css`, and the other `z-*.css` files.

**`static/`:**
- Purpose: Files copied verbatim into `public/` at build time.
- Contains: Raw CSS not in the asset pipeline, JavaScript, images, presentation decks, generated CV.
- Key files: `static/css/cv.css`, `static/js/hero-type.js`, `static/images/me.png`, `static/terminal.css`. Generated: `static/cv.html`, `static/cv.pdf`.

**`data/`:**
- Purpose: Structured non-Markdown inputs.
- Contains: `home.toml` (homepage landing content, available as `.Site.Data.home`) and `skills.toml` (categorized skill lists used by the CV generator and available as `.Site.Data.skills`).
- Key files: `data/home.toml`, `data/skills.toml`.

**`scripts/`:**
- Purpose: Pre-build / utility scripts run from the Makefile.
- Contains: Python 3 stdlib-only generators.
- Key files: `scripts/generate_cv.py`.

**`archetypes/`:**
- Purpose: Frontmatter templates used when scaffolding new content via `hugo new`.
- Contains: `default.md` (date, draft, title).
- Key files: `archetypes/default.md`.

**`.github/workflows/`:**
- Purpose: GitHub Actions CI/CD definitions.
- Contains: `hugo.yml` — single workflow that builds the CV, builds the site, and deploys to GitHub Pages.
- Key files: `.github/workflows/hugo.yml`.

**`public/` and `resources/`:**
- Purpose: Hugo build outputs and asset cache.
- Contains: Generated HTML/CSS/JS (`public/`), processed asset cache (`resources/_gen/`).
- Key files: None (gitignored, regenerated on every build).

**`.planning/`:**
- Purpose: GSD planning artifacts (plans, phases, codebase maps).
- Contains: `codebase/` — analysis docs consumed by other GSD commands.
- Key files: `ARCHITECTURE.md`, `STRUCTURE.md`, `STACK.md`, etc.

## Key File Locations

**Entry Points:**
- `hugo.toml`: Hugo site config (theme, base URL, languages, menu, markup options).
- `Makefile`: Developer/CI command surface (`serve`, `build`, `new`, `cv`, `cv-pdf`, `update-theme`).
- `.github/workflows/hugo.yml`: CI build + Pages deploy.

**Configuration:**
- `hugo.toml`: Site-wide config.
- `.gitmodules`: Theme submodule pin.
- `.gitignore`: Excludes `public/`, `resources/_gen/`, `static/cv.html`, `static/cv.pdf`, Hugo binaries, `.hugo_build.lock`.
- `.nojekyll` (and `.nojeckyll` at repo root): Disable Jekyll on GitHub Pages.

**Core Logic:**
- `scripts/generate_cv.py`: Markdown → HTML CV generator.
- `layouts/index.html`: Homepage landing template (hero typewriter, stats band, CTAs), driven by `data/home.toml`.
- `layouts/404.html`: Custom terminal-styled 404 page.
- `layouts/partials/extended_head.html`: Theme `<head>` extension point (KaTeX, structured data, analytics).
- `layouts/partials/structured_data.html`: Schema.org JSON-LD (WebSite/Person/ProfilePage/BlogPosting).
- `static/js/hero-type.js`: Hero role-line typewriter (progressive enhancement, reduced-motion aware).
- `layouts/_default/_markup/render-link.html`: Custom link rendering (external links open in new tab).
- `layouts/shortcodes/x.html`: Twitter/X embed shortcode.

**Content:**
- `content/_index.md`: Homepage.
- `content/about/_index.md`: About page (also a CV input).
- `content/previous-work.md`: Work history (also a CV input).
- `content/posts/<slug>/index.md`: Each blog post (Hugo leaf bundle).

**Styling:**
- `assets/css/z-base.css`: Shared CSS tokens (`--c-text-*`, `--c-hairline`, `--c-error`), the `blink` cursor keyframe, and the `.sr-only` utility, reused by the other `z-*.css` files.
- `assets/css/z-home.css`, `assets/css/z-404.css`, and the other `z-*.css` files: Page/component styles (asset pipeline, picked up by `css/*.css`).
- `static/terminal.css`: Raw CSS shipped to `public/`.
- `static/css/cv.css`: Inlined into generated `static/cv.html`.

**Testing:**
- None present. There are no test files, test runner config, or CI test step in this repo.

## Naming Conventions

**Files:**
- Markdown content uses lowercase-kebab-case filenames and slugs (`how-to-exploit-ethereum-in-a-virtual-environment`).
- Posts that are leaf bundles use `index.md` as the entry file inside a slug-named directory.
- Co-located post images use plain descriptive names (`image.png`, `image 1.png`); spaces are tolerated.
- Templates and partials are lowercase with underscores (`extended_head.html`, `render-link.html`); shortcodes are short identifiers (`x.html`).
- Generated artifacts use `.html` / `.pdf` extensions and are gitignored.

**Directories:**
- Content directories mirror URL slugs: `content/posts/<slug>/` → `/posts/<slug>/`.
- Theme override directories mirror theme structure exactly (`layouts/_default/_markup/...`, `layouts/partials/...`, `layouts/shortcodes/...`).
- Custom CSS lives under `assets/css/` as `z-*.css` files, picked up by the theme's `css/*.css` asset pipeline.

**Frontmatter:**
- TOML format only (`+++` fences). Standard keys: `title`, `date`, `draft`, `slug`, `cover`, `tags`, `keywords`, `description`, `showFullContent`, `authorTwitter`, `math`.

## Where to Add New Code

**New blog post:**
- Run `make new POST="My Post Title"` → creates `content/posts/my-post-title/index.md` from `archetypes/default.md`.
- Add images alongside `index.md` in the same directory; reference them with relative paths.
- If the post uses math, set `math = true` in frontmatter.

**New non-post page (e.g. `/projects/`):**
- Create `content/projects/_index.md` (or `content/projects.md`) with TOML frontmatter (`title`, `slug`, `draft = false`).
- For menu inclusion, add an entry under `[[languages.en.menu.main]]` in `hugo.toml`.

**New theme override (template tweak):**
- Identify the file in `themes/terminal/layouts/...` you want to change.
- Create a same-path file under `layouts/...` and place the override there. Do not edit the theme directly.

**New shortcode:**
- Add `layouts/shortcodes/<name>.html`. Follow the pattern in `layouts/shortcodes/x.html`. Invoke from Markdown as `{{< name arg="..." >}}`.

**New CSS rule:**
- For component/page styles that layer on top of the theme's compiled CSS: add a `z-<name>.css` file under `assets/css/` (e.g. `assets/css/z-contact.css`). Every `assets/css/*.css` file is picked up, minified and fingerprinted by the theme's asset pipeline; the `z-` prefix keeps these custom additions grouped and last in load order. Put shared tokens/animations/utilities in `assets/css/z-base.css` (e.g. the color tokens, the `blink` keyframe, and the `.sr-only` screen-reader-only utility already live there).
- For raw CSS shipped as-is: add to `static/css/` and reference it from a partial loaded in `layouts/partials/head.html`.

**New static asset (image, PDF, slide deck):**
- Drop into `static/<category>/...`; it will be served at `/<category>/...`. Reuse existing categories where possible (`static/images/`, `static/presentations/`).

**New CV input field:**
- For skill categories: edit `data/skills.toml` (sections become headings).
- For narrative sections: edit `content/about/_index.md` or `content/previous-work.md`. The CV generator reads both directly.
- For layout/styling of the CV: edit `static/css/cv.css` (it is inlined into `static/cv.html` at generation time).
- Re-run `make cv` (HTML) or `make cv-pdf` (HTML + PDF) to regenerate.

**New build/automation step:**
- Add a `.PHONY` target to `Makefile` and document it in the help block (`## description` after the colon is auto-extracted by `make help`).
- Wire into CI by adding a step to `.github/workflows/hugo.yml` before the `hugo` build step.

**Utilities / scripts:**
- Place under `scripts/`. Keep stdlib-only where possible (matches the precedent set by `scripts/generate_cv.py`); CI does not install Python dependencies.

## Special Directories

**`themes/terminal/`:**
- Purpose: Vendored Hugo theme.
- Generated: No.
- Committed: As a submodule pointer only (the actual files come from the upstream repo at the pinned SHA).

**`public/`:**
- Purpose: Hugo build output served by GitHub Pages.
- Generated: Yes (every `hugo` invocation).
- Committed: No (gitignored).

**`resources/`:**
- Purpose: Hugo asset cache (`resources/_gen/` for processed images, etc.).
- Generated: Yes.
- Committed: `resources/_gen/` is gitignored; the `resources/` parent may exist with placeholder content.

**`static/cv.html`, `static/cv.pdf`:**
- Purpose: Generated CV artifacts served at `/cv.html` and `/cv.pdf`.
- Generated: Yes (`make cv` / `make cv-pdf`).
- Committed: No (gitignored).

**`.planning/`:**
- Purpose: GSD workflow artifacts.
- Generated: By GSD commands (codebase mappers, planners, executors).
- Committed: Typically yes, so future runs and other contributors share context.

**`vector-outline.html`:**
- Purpose: Standalone HTML file at repo root (not under `static/`, so Hugo does not copy it into `public/`). Likely an experimental/standalone artifact.
- Generated: No.
- Committed: Yes.
